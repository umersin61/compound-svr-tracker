Method & reliability

Activation windows (the key correction). Liquidations are counted only from the block each comet's price feeds were switched to the SVR aggregator on-chain (verified via the Comet proxy Upgraded event whose implementation first points at the SVR feed): Ethereum cUSDTv3 2025-09-18, cUSDCv3 2025-10-23 (proposal 488); Base cAEROv3 2026-02-03, cUSDCv3 2026-04-10; Arbitrum cUSDTv3 & cWETHv3 2026-02-12, cUSDCv3 2026-04-10. This dropped 468 pre-activation liquidations — including 9 that had been falsely credited as SVR recaptures on cUSDCv3 before its feed switch.

Liquidations & available incentive. Every AbsorbDebt event per comet (value liquidated = on-chain usdValue). Available OEV = Σ over seized collateral of collateral_usd × storeFrontPriceFactor × (1 − liquidationFactor), read on-chain (storeFront 0.60, 0.70 on WETH comets). Theoretical-max discount, so "leaked" is an upper bound. Fully reliable inputs.

Recapture, in USD at recapture-time. A liquidation is SVR-routed when the Chainlink collector Safe 0x149B…14e5 received an ETH bid in the same block (reconciles with Chainlink's reported SVR counts). The verified anchor is the 354.49 ETH delivered to the DAO recipient (gross ≈ 644 ETH at 55/45). Per-row DAO/Chainlink amounts allocate that verified total across SVR-routed mainnet liquidations by available incentive, then value the ETH at the liquidation month's ETH price — so every column is in USD. Per-row figures are estimates; the ETH total is exact. The collector is shared across Aave/Venus/Compound, so exact per-liquidation isolation isn't possible on-chain.

Base/Arbitrum. Activated but the collector shows only dust there — no SVR recapture flowing on-chain yet, so those rows carry available incentive but no recapture. Most L2 liquidations are sub-dollar test absorbs.
